  • This DigitalOcean tutorial was used to setup an api endpoint to use with a GitHub webhook and automatically pull from GitHub whenever the repo is pushed to.
  • Various pages of the MongoDB docs and docs of the NodeJS driver were instrumental in setting up the MongoDB database and parts of the API that interact with it.
  • This zellwk.com blog post was used for ideas and inspiration regarding the account and login system.
  • This coderocketfuel.com article was used for the specifics of hashing stored passwords in the database along with a stackoverflow post that the reference to has since been lost.
  • This w3schools article was used to setup verification emails for the /login/verify API endpoint.
  • The Bulma docs have been utilized as reference for working with Bulma components and classes.
  • The MDN web docs for Fetch were used for setting up fetch requests.
  • Bits and pieces of the preview of the cors in action book were used to figure out the server side of CORS compliance.
  • The Factorio website and the Factorio game itself are used as the basis for the app's entire concept as well as look and style.
